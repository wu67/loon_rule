# 自用规则

用来自定义规则文件, 其中 proxy和 direct为自己定义, 另外两个从第三方仓库获取并处理整合.

- proxy: 代理规则, 走代理的流量
- direct: 直连规则, 直连的流量
- reject: 拒绝规则(广告), 直接拒绝的流量
- ip: ip直连规则, 直连的流量. 相比原本的规则, 增加了no-resolve选项, 避免DNS污染

#### 核心思路是 广告拒绝 国内直连, 其他(兜底)代理.

#### 配置顺序是

1. 你自己的补充拒绝规则(若有)
1. reject
1. 公共仓库的拒绝规则(可以省略)
1. 你自己的补充代理规则(若有)
1. proxy 这是用来强制部分 可以直连的域名 走代理, 例如苹果、谷歌cn
1. 公共仓库的代理规则(可以省略)
1. 你自己的补充直连规则(若有)
1. direct (看情况, 目前是空的, 我用公共的还没发现漏掉的需要直连的)
1. 公共仓库的直连规则
1. ip 直连
